# Invoice Module — Implementation Plan

## সংক্ষিপ্ত বিবরণ

ZamZam ERP-তে একটি পূর্ণাঙ্গ Invoice Module তৈরি করা হবে। Invoice দুইভাবে তৈরি করা যাবে:
1. **Sales Order থেকে generate** — SO items অটো-copy হবে
2. **Standalone** — যেকোনো Customer-এর জন্য সরাসরি তৈরি করা যাবে

Payment sync হবে SO-এর `paid_bdt`/`due_bdt` থেকে (যখন SO linked), Invoice status অটো-আপডেট হবে। Print/PDF সাপোর্ট থাকবে।

## ব্যবসায়িক নিয়ম (Business Rules)

1. **Invoice তৈরির পদ্ধতি:**
   - SO থেকে: SO-এর `items`, `subtotal_bdt`, `discount_bdt`, `delivery_charge_bdt`, `total_bdt`, `customer_id` অটো-কপি
   - Standalone: Customer সিলেক্ট করে manually items যোগ করতে হবে

2. **Invoice items editable:** Create/Edit উভয় ক্ষেত্রেই qty ও price পরিবর্তন করা যাবে

3. **Payment sync (SO-linked invoice):**
   - `Invoice::syncFromSalesOrder()` method: SO-এর `paid_bdt`, `due_bdt` দেখে invoice-এর `paid_bdt`, `due_bdt` আপডেট করবে
   - SO-এর `receivePayment` সম্পন্ন হলে linked invoice sync হবে (Service layer-এ)

4. **Invoice status auto-update:**
   ```
   draft     → যখন তৈরি হয়
   issued    → manually issue করা হলে
   partial   → 0 < paid_bdt < total_bdt
   paid      → paid_bdt >= total_bdt
   overdue   → issued + due_date < today + due_bdt > 0
   cancelled → manually cancel করা হলে
   ```

5. **Invoice number format:** `INV-{YEAR}-{0001}` (SalesOrder-এর SO-{YEAR}-{0001} pattern অনুসরণ)

6. **Print:** Invoice Show page-এ `@media print` CSS দিয়ে print-friendly view, "Print" বাটনে `window.print()` call

## তৈরি করতে হবে এমন ফাইলসমূহ

### Backend (8টি ফাইল)

#### 1. `InvoiceItem` Model
**Path:** `app/Models/Sales/InvoiceItem.php`

```php
namespace App\Models\Sales;
- fillable: invoice_id, product_id, product_variant_id, quantity, unit_price_bdt, subtotal_bdt
- auto-calculates subtotal_bdt on saving (quantity * unit_price_bdt)
- belongsTo: Invoice, Product, ProductVariant
```

#### 2. `Invoice` Model — Update
**Path:** `app/Models/Sales/Invoice.php` (existing — extend করতে হবে)

নতুন যোগ করতে হবে:
- `syncFromOrder(SalesOrder $order)` method — SO-এর paid/due থেকে invoice আপডেট + status recalculate
- `recalculateStatus()` method — paid/due/due_date দেখে status set করা
- `generateInvoiceNo()` static method
- `scopeForCustomer()`, `scopeByStatus()` scopes

#### 3. `StoreInvoiceRequest`
**Path:** `app/Http/Requests/Sales/StoreInvoiceRequest.php`

```
Fields:
- sales_order_id: nullable|integer|exists:sales_orders,id
- customer_id: required|integer|exists:customers,id
- issue_date: required|date
- due_date: nullable|date|after_or_equal:issue_date
- notes: nullable|string|max:1000
- discount_bdt: nullable|numeric|min:0
- delivery_charge_bdt: nullable|numeric|min:0
- items: required|array|min:1
- items.*.product_id: required|integer|exists:products,id
- items.*.product_variant_id: nullable|integer|exists:product_variants,id
- items.*.quantity: required|integer|min:1
- items.*.unit_price_bdt: required|numeric|min:0
```

#### 4. `UpdateInvoiceRequest`
**Path:** `app/Http/Requests/Sales/UpdateInvoiceRequest.php`

StoreInvoiceRequest-এর মতো, `sometimes` modifier সহ। Status পরিবর্তনের জন্য আলাদা endpoint।

#### 5. `InvoiceService`
**Path:** `app/Services/Sales/InvoiceService.php`

```php
Methods:
- generateInvoiceNo(): string
- createInvoice(array $data, int $createdBy): Invoice
  → DB::transaction: Invoice create + InvoiceItem create + recalculateTotals + recalculateStatus
- createFromSalesOrder(SalesOrder $order, array $extra, int $createdBy): Invoice
  → SO থেকে items copy + SO fields inherit
- updateInvoice(Invoice $invoice, array $data): Invoice
  → items full sync (delete + recreate) + recalculate
- issueInvoice(Invoice $invoice): Invoice
  → status = 'issued', validates must be draft
- cancelInvoice(Invoice $invoice): Invoice
- syncPaymentFromOrder(Invoice $invoice): Invoice
  → SO-এর paid_bdt দেখে invoice paid/due আপডেট + recalculateStatus
```

#### 6. Web `InvoiceController` (Inertia)
**Path:** `app/Http/Controllers/Web/Admin/Sales/InvoiceController.php`

```php
Methods:
- index(Request): Response — filters: status, customer_id, search, date range
- create(Request): Response — optional ?sales_order_id param, loads SO data if provided
- show(Invoice): Response
- edit(Invoice): Response|RedirectResponse — only draft/issued
```

Inertia props for create/edit:
- `invoice` (edit only), `order` (if SO-linked), `customers`, `salesOrders` (for standalone)

#### 7. API `InvoiceController`
**Path:** `app/Http/Controllers/Api/V1/Sales/InvoiceController.php`

```php
Endpoints:
- GET /api/v1/invoices — list with filters
- POST /api/v1/invoices — create (standalone or from SO via sales_order_id field)
- GET /api/v1/invoices/{invoice} — show
- PUT /api/v1/invoices/{invoice} — update
- ~~DELETE~~ — নেই (cancelled status ব্যবহার করা হবে)
- POST /api/v1/invoices/{invoice}/issue — status → issued
- POST /api/v1/invoices/{invoice}/cancel — status → cancelled
- POST /api/v1/invoices/{invoice}/sync-payment — SO payment থেকে sync
```

### Frontend (4+1 Vue Pages)

#### 1. `Index.vue`
**Path:** `resources/js/Pages/Sales/Invoices/Index.vue`

- Status tabs: All / Draft / Issued / Partial / Paid / Overdue / Cancelled (count badges সহ)
- Search: invoice_no, customer name, phone
- Filters: customer, date range
- Table columns: Invoice No, Customer, Issue Date, Due Date, Total, Paid, Due, Status, Actions
- Actions: View, Edit (draft/issued only), Print
- Dark mode সম্পূর্ণ সাপোর্ট

#### 2. `Create.vue`
**Path:** `resources/js/Pages/Sales/Invoices/Create.vue`

দুই মোড (URL param দ্বারা নিয়ন্ত্রিত):

**Mode A — SO থেকে:** `?sales_order_id=X` থাকলে
- SO info readonly display (order_no, customer, totals)
- Items SO থেকে pre-populated, qty/price editable
- Discount ও delivery charge editable

**Mode B — Standalone:**
- Customer search/select dropdown
- Items manually add (product search + qty + price)
- Totals auto-calculate

Common fields (উভয় mode):
- Issue Date (default: today)
- Due Date (optional)
- Notes
- Discount BDT
- Delivery Charge BDT
- Live totals preview (subtotal, discount, delivery, total)

Submit: `POST /api/v1/invoices` → redirect to `invoices.show`

#### 3. `Edit.vue`
**Path:** `resources/js/Pages/Sales/Invoices/Edit.vue`

- Create.vue থেকে pre-populate সব fields
- SO-linked হলে SO info show করবে (customer lock)
- Submit: `PUT /api/v1/invoices/{id}` → redirect to `invoices.show`
- Cancel: back to Show page (BackButton pattern)

#### 4. `Show.vue`
**Path:** `resources/js/Pages/Sales/Invoices/Show.vue`

Layout (2-column grid):

**Left column:**
- Invoice metadata card: invoice_no, status badge, issue_date, due_date, created_by
- Customer info card: name, business_name, phone, email
- Linked SO card (যদি SO-linked থাকে): SO number, status → link to SO show page
- Financial summary card: subtotal, discount, delivery, total, paid (emerald), due (red/green)
- Action buttons:
  - Edit (draft/issued only)
  - Issue Invoice (draft only)
  - Cancel (draft/issued only)
  - Sync Payment from SO (SO-linked + issued/partial only)
  - Print button

**Right column:**
- Invoice Items table: Product, Variant, Qty, Unit Price, Subtotal

**Print area (`@media print` CSS):**
- Header: Company name / logo placeholder, Invoice No, dates
- Customer info
- Items table (full width)
- Totals section
- Notes (যদি থাকে)
- Non-print elements hide করা (`print:hidden` class)

## গুরুত্বপূর্ণ Constraints

1. **AGENTS.md rules:** Icon rules, dark mode, color tokens, button loading state — সব mandatory
2. **Edit.vue** সবসময় Create.vue-এর সাথেই তৈরি করতে হবে
3. **API routes ordering:** static paths (`/invoices/trashed`) wildcard (`/invoices/{invoice}`) এর আগে
4. **Print CSS:** `@media print` এ navigation/sidebar hide, invoice content full-width
5. **`sales_order_id` nullable:** Invoice standalone হলে NULL থাকবে
6. **No delete:** Invoice delete করা যাবে না — শুধু `cancelled` status ব্যবহার করতে হবে (`deleted_at` column নেই, soft delete নেই)

# UI Design System Overhaul Plan
**ZamZam ERP — Design Rules, 3D Icons, Dark Mode, Theme Customization**

---

## Goals Summary

| Feature | Decision |
|---------|---------|
| 3D Icons | সব জায়গায় (module-level: PNG, inline/action: CSS 3D effect) |
| Color storage | Database per-user (`user_preferences` table) |
| Dark mode | Manual toggle (TopBar) + OS preference auto-detect |
| Color options | Predefined 6-8 swatches **+** full hex color picker |

## Phase 1 — Design System Rules (AGENTS.md Update)

AGENTS.md-এ নতুন section যোগ করতে হবে:

### Icon Rules
```
## Icon Usage Rules

### Module/Section Level Icons (≥ w-8)
Use <ThreeDIcon name="..." size="md|lg|xl" /> (Icons8 Fluency PNG)
- Sidebar navigation items
- Page headers (h1 পাশে)
- Section headers
- Stat cards
- Dashboard widgets
- Empty state illustrations

### Inline/Action Level Icons (< w-6)
Use Lucide icons with class="icon-3d" for 3D CSS effect:
- Buttons (edit, delete, save, cancel)
- Table action columns
- Form field prefix/suffix
- Breadcrumb separators
- Badge/tag icons
- Dropdown menu items
```

### Dark Mode Rules
```
## Dark Mode Rules
Every component MUST include dark: variant for:
- Backgrounds: dark:bg-slate-900 / dark:bg-slate-800 / dark:bg-slate-700
- Text: dark:text-slate-100 / dark:text-slate-300 / dark:text-slate-400
- Borders: dark:border-slate-700 / dark:border-slate-600
- Inputs: dark:bg-slate-800 dark:border-slate-600 dark:text-slate-100
- Cards: dark:bg-slate-800 dark:border-slate-700
- Hover: dark:hover:bg-slate-700
```

### Color Token Rules
```
## Color Token Rules
Hard-coded hex (#6366f1) ব্যবহার নিষেধ। সর্বদা Tailwind token:
- primary-50 through primary-700 → theme-aware (runtime-variable)
- brand-600/700 → BD Green accent (static)
- slate-* → neutral surfaces
```

## Phase 2 — Backend: User Preferences

### 2.1 Migration
File: `database/migrations/2026_05_18_XXXXXX_create_user_preferences_table.php`

```php
Schema::create('user_preferences', function (Blueprint $table) {
    $table->id();
    $table->foreignId('user_id')->constrained()->cascadeOnDelete();
    $table->string('theme_color', 20)->default('indigo');   // preset name
    $table->string('accent_hex', 7)->nullable();             // custom hex override
    $table->boolean('dark_mode')->default(false);
    $table->timestamps();
    $table->unique('user_id');
});
```

### 2.2 Model
File: `app/Models/UserPreference.php`

- `$fillable`: `user_id`, `theme_color`, `accent_hex`, `dark_mode`
- `$casts`: `dark_mode` → boolean
- Relationship: `user()` belongsTo User

### 2.3 User Model Update
`app/Models/User.php` → add `preferences()` hasOne UserPreference

### 2.4 API Endpoint
Route: `PUT /api/v1/user/preferences`
File: `app/Http/Controllers/Api/V1/UserPreferenceController.php`
- `update()`: upsert preference for auth user
- Returns updated preferences

Route file: `routes/modules/user.php` (or add to existing auth routes)

### 2.5 HandleInertiaRequests Update
`app/Http/Middleware/HandleInertiaRequests.php` → share:

### 3.3 app.css Update — Core Changes

**Dark mode variant (Tailwind v4):**
```css
@custom-variant dark (&:where(.dark, .dark *));
```

**Inline theme — runtime-switchable primary:**
```css
@theme inline {
  --color-primary-50:  var(--brand-50);
  --color-primary-100: var(--brand-100);
  --color-primary-500: var(--brand-500);
  --color-primary-600: var(--brand-600);
  --color-primary-700: var(--brand-700);
}
```

**Per-theme CSS variable blocks:**
```css
:root, .theme-indigo {
  --brand-50: #eef2ff; --brand-500: #6366f1;
  --brand-600: #4f46e5; --brand-700: #4338ca;
}
.theme-blue    { --brand-50: #eff6ff; --brand-500: #3b82f6; ... }
.theme-violet  { --brand-50: #f5f3ff; --brand-500: #8b5cf6; ... }
.theme-emerald { --brand-50: #ecfdf5; --brand-500: #10b981; ... }
.theme-orange  { --brand-50: #fff7ed; --brand-500: #f97316; ... }
.theme-rose    { --brand-50: #fff1f2; --brand-500: #f43f5e; ... }
.theme-sky     { --brand-50: #f0f9ff; --brand-500: #0ea5e9; ... }
.theme-amber   { --brand-50: #fffbeb; --brand-500: #f59e0b; ... }
```

**Custom hex override (via JS):**
```js
// When user picks custom hex, ThemeStore runs:
document.documentElement.style.setProperty('--brand-500', hexValue)
// (computed lighter/darker shades automatically)
```

**Dark mode base:**
```css
.dark { color-scheme: dark; }
.dark body { background: theme(--color-slate-900); }
```

**3D icon CSS utility:**
```css
.icon-3d {
  filter: drop-shadow(0 2px 4px rgba(0,0,0,0.25))
          drop-shadow(0 1px 2px rgba(0,0,0,0.15));
  transition: filter 0.2s, transform 0.2s;
}
.icon-3d:hover {
  filter: drop-shadow(0 4px 8px rgba(0,0,0,0.30))
          drop-shadow(0 2px 4px rgba(0,0,0,0.20));
  transform: translateY(-1px);
}
```

## Phase 7 — Interactive Layout Enhancements

### 7.1 Page Transitions
Inertia progress bar already exists. Add Vue transition on `<main>` slot:
```vue
<Transition name="page-fade" mode="out-in">
  <main :key="$page.url" class="flex-1 overflow-y-auto">
    <div class="p-6 max-w-[1600px] mx-auto">
      <slot />
    </div>
  </main>
</Transition>
```
```css
.page-fade-enter-active, .page-fade-leave-active { transition: opacity 0.15s, transform 0.15s; }
.page-fade-enter-from, .page-fade-leave-to { opacity: 0; transform: translateY(4px); }
```

### 7.2 Sidebar Animation
SidebarGroup accordion: replace instant toggle with `<Transition name="slide-down">` + `max-height` animation.

### 7.3 Card Hover Effects  
StatCard already has `hover:-translate-y-0.5`. Extend to other list/table rows:
```
hover:bg-slate-50 dark:hover:bg-slate-700/50 transition-colors duration-150
```

### 7.4 Button Loading States
Standard button pattern:
```vue
<button :disabled="loading" class="... disabled:opacity-50 disabled:cursor-not-allowed">
  <LoaderIcon v-if="loading" class="w-4 h-4 animate-spin icon-3d" />
  <Icon3D v-else name="Save" size="sm" />
  {{ loading ? 'Saving...' : 'Save' }}
</button>
```

## Notes

- Tailwind v4 uses `@theme inline` for runtime-variable color tokens — verified compatible.
- `@custom-variant dark` enables `dark:` class-based scoping without `prefers-color-scheme` forcing.
- Icons8 Fluency icons are already in `/public/icons/fluency/` — only ICON_MAP expansion needed, no new assets required for known icons. Unknown action icons need PNG downloads.
- `@vueuse/core` (v14) already installed — `useColorMode()` or `usePreferredDark()` can be used for OS detection.
